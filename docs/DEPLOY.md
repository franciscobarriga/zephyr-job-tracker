# Deploying Zephyr

The web app is containerized — any platform that runs a `Dockerfile` works.
The scraper continues to run on **GitHub Actions** (cron), so the deployed
web app does NOT need to scrape on its own.

## Render (free tier)

1. Sign in at https://render.com.
2. **New + → Blueprint** → pick `franciscobarriga/zephyr-job-tracker`.
   Render reads `render.yaml` and provisions a free web service.
3. It will prompt for the secret env vars (`sync: false` in the blueprint):
   - `SUPABASE_URL`
   - `SUPABASE_ANON_KEY`
   - `SUPABASE_SERVICE_KEY`
   - `ANTHROPIC_API_KEY`
   - `APP_BASE_URL` (set this to the Render URL once known, e.g.
     `https://zephyr.onrender.com`; you can edit it after first deploy)
   - `SECRET_KEY` is auto-generated by Render.
4. First build takes ~5 minutes (Playwright image).

Free-tier caveats:
- Service sleeps after 15 min idle; first request after that takes ~50s.
- 512 MB RAM. The in-app "Run scraper" button launches Chromium and may be
  memory-tight; the GitHub Actions scraper is unaffected and keeps the board
  populated regardless.

After the URL is known, update Supabase Auth (Site URL + redirect allowlist)
to the Render URL so password-reset links point to the right place.

## Railway (paid)

Railway works identically (`railway.toml` is included) but the free trial is
time-limited. New project → Deploy from GitHub repo → set the same env vars →
Generate Domain.

## Local docker run

```bash
docker build -t zephyr .
docker run --rm -p 8000:8000 --env-file .env zephyr
```

## What the deployed web app does (and doesn't)

- ✅ Serves the dashboard, jobs board, resume upload, search config UI
- ✅ Calls Anthropic for resume tailoring / on-demand AI features
- ✅ Reads/writes Supabase
- ❌ Does NOT run the scraper. Scraping happens in GitHub Actions
  (`.github/workflows/scraper.yml`) on a cron, writes to the same Supabase
  project. Cleanup runs separately on a daily cron.

This keeps the web container small, scraper-runs auditable in Actions logs,
and removes the headless-browser failure mode from request handling.
